## 1. Функциональность, которую можно заменить на публичную платформу

В проекте есть функциональность, связанная с картой, например, пользователь может указать место потери или находки, посмотреть объявления на карте, выбрать район или радиус поиска. Эту часть не обязательно разрабатывать полностью самостоятельно. Вместо собственной карты можно использовать готовую публичную платформу, например Яндекс Карты API.

### Как именно это можно использовать

Можно реализовать:

- отображение карты Москвы;

- выбор точки на карте при создании объявления;

- показ маркеров объявлений;

- поиск объявлений в выбранном районе или радиусе;

- перевод введённого адреса в точку на карте.

Таким образом, можно не тратить ресурсы на разработку собственной карты, а использовать уже готовую платформу. 

## 2. Стратегия платформизации 

Проект может быть не только обычным приложением, но и готовым сервисом для организаций. Например, торговые центры, транспортные компании, аэропорты, приюты и бюро находок могут его использовать как свою цифровую систему для работы с потерянными и найденными вещами.

### Основные клиенты:

1. **Торговые центры**
В ТЦ часто теряют ключи, кошельки, документы, сумки, одежду. Сейчас такие находки могут храниться на стойке информации или фиксироваться вручную. 

2. **Транспортные компании**
Например, метро, автобусы, вокзалы, РЖД. В транспорте часто оставляют вещи, и пользователям неудобно искать информацию в разных местах.

3. **Аэропорты**
В аэропортах много потерянных вещей, и важно быстро связать владельца и его вещь.

4. **Приюты и службы, связанные с животными**
Они могут публиковать информацию о найденных животных, чтобы владельцы быстрее их находили.

5. **Городские сервисы**
Например, город может использовать Lost&Found как единый сервис для поиска потерянных вещей и животных.

### Как организации будут использовать сервис

Для организаций можно сделать отдельный кабинет.

Допустим, сотрудник торгового центра нашёл кошелёк. Он заходит в кабинет организации, создаёт объявление, указывает категорию, время и основные признаки. После этого объявление появляется в общей базе Lost&Found.

Пользователь, который потерял кошелёк, заходит в сервис, ищет по месту и времени, видит объявление от торгового центра и откликается. Дальше стороны проходят проверку владельца через контрольные вопросы. Если владелец подтверждён, организация передаёт вещь и закрывает объявление.

### Модель бизнеса

Для проекта больше всего подходит модель B2B2C.

- Сервис продаётся организациям;

- организации используют его для своих процессов;

- конечные пользователи ищут и возвращают вещи через этот сервис.

Для городских сервисов также можно рассматривать модель B2G2C, где Lost&Found работает не с бизнесом, а с городской организацией или государственными сервисами.

### Стратегия монетизации

Основной вариант монетизации это периодическая подписка для организаций.

Организация может платить каждый месяц или каждый год за доступ к сервису. В подписку может входить:

- кабинет организации;

- размещение объявлений о находках;

- обработка обращений пользователей;

- статусы объявлений;

- история обращений;

- отчёты по найденным и возвращённым вещам.

Можно сделать несколько тарифов:

1. **Базовый тариф**
Для небольших организаций. Включает ручное размещение объявлений и работу с обращениями.

2. **Расширенный тариф**
Для торговых центров, приютов, небольших транспортных организаций. Может включать кабинет сотрудников, статистику и больше возможностей по управлению объявлениями.

3. **Большой тариф**
Для аэропортов, транспортных компаний и городских сервисов. Включает интеграцию с их внутренними системами и отдельные настройки.
