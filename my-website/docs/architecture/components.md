## Назначение

Компонентная модель показывает основные части системы и зоны ответственности между frontend, backend, хранением данных и внешними сервисами.

## Компоненты

В разделе фиксируются web-клиент, API backend, база данных, файловое хранилище, модуль owner-check и модерация.

## Взаимодействия

Здесь описываются синхронные REST-вызовы, асинхронный канал для owner-check и связи с внешними сервисами.

## Ответственность компонентов

Для каждого компонента указываются ответственность, входные и выходные данные, зависимости и ограничения.
