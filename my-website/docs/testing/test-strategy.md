## Назначение

Стратегия тестирования описывает, какие сценарии MVP проверяются перед релизом и какие риски закрываются тестами.

## Область тестирования

В область входят пользовательские сценарии, API, owner-check, публикация объявлений и работа с жалобами.

## Виды тестирования

Используются функциональное тестирование, проверка API, негативные сценарии, smoke-тесты сборки и регрессионные проверки.

## Критерии готовности

Ключевые Use Case должны иметь приёмочные проверки и проходить без критических дефектов.
