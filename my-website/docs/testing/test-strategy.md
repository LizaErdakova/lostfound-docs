## Назначение

Документ описывает подход к проверке функциональности Lost&Found MVP.

## Область тестирования

В разделе фиксируются пользовательские сценарии, API, owner-check, публикация объявлений и работа с жалобами.

## Виды тестирования

Функциональное тестирование, проверка API, негативные сценарии, smoke-тесты сборки и регрессионные проверки.

## Критерии готовности

Функциональность считается готовой, когда ключевые Use Case имеют приемочные проверки и проходят без критических дефектов.
